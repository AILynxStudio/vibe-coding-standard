# 贡献指南

感谢你考虑为 Vibe Coding 工业级开发标准体系做出贡献！本指南将帮助你了解如何参与。

## 贡献方式

- **报告 Bug / 提出建议**：在 GitHub Issues 中提交，描述清楚使用场景和遇到的问题。
- **改进标准**：如果你在实践中发现了更好的流程或规则，欢迎提交 PR。
- **补充案例**：如果你有完整的实战案例（含 AI 对话、代码、阶段产出），欢迎提交到 `cases/` 目录。
- **完善图表**：如果你能用绘图工具重新绘制更美观的图表，欢迎覆盖 `charts/` 中的图片。

## Pull Request 流程

1. Fork 本仓库。
2. 创建你的特性分支 (`git checkout -b feature/your-feature`)。
3. 修改内容。
4. 确保新增的 markdown 文件符合标准格式（参考已有文件）。
5. 提交时清晰描述修改内容。
6. 发起 Pull Request，等待审核。

## 提交规范

- 标准文件修改：请附带修改原因和实际使用效果。
- 工具包修改：请确保与第一部分标准保持一致，并更新版本号。
- 案例补充：请按现有案例的结构撰写，包含"案例档案"、"前置准备"、"完整步骤"、"常见问题"。

## 版本管理

- 主版本号（X.0.0）：标准体系重大调整或场景增删。
- 次版本号（0.X.0）：新增案例、工具包微调。
- 修订号（0.0.X）：文字修正、格式调整。

## 行为准则

请保持友善、建设性，我们推崇理性讨论。

## 许可证

贡献内容将在 MIT 许可证下发布。
