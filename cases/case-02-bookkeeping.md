# 案例2：桌面 AI 客户端——做一个人记账网页

> **难度**：★★（需要会在电脑上创建文件夹，保存文件）
> **工具包**：TK-4（桌面客户端提示词）+ TK-5（8阶段Task模板）
> **你需要**：电脑上安装了 ChatGPT 桌面版 或 Claude 桌面版 或 Kimi 桌面版（任选一个）；一个代码编辑器（推荐下载 VS Code，但用记事本也行）
> **你将做出**：一个可以记录收入和支出、数据能保存到浏览器的个人记账网页

## 前置准备

1. 在电脑上新建一个文件夹，命名为 `my-account-book`（放在桌面或文档里都行）。
2. 打开你的桌面 AI 客户端。
3. 打开第〇章 TK-5 模板，复制到记事本里，作为你的进度表。

## 第 1 步：发送 TK-4 给 AI（1 分钟）

打开桌面 AI 客户端，将第〇章 TK-4 的完整内容复制，作为第一条消息发送给 AI。

AI 会回复确认已理解规则。如果它回复很简短（如“好的，明白了”），这就够了，继续下一步。

## 第 2 步：需求梳理（阶段 1 简化版，5 分钟）

告诉 AI：

> 阶段1：我要做一个个人记账网页。核心功能：记录收入和支出条目，显示总余额，数据保存在浏览器本地存储中。次要功能：按日期筛选、分类统计（餐饮/交通/购物/其他）。扩展功能：导出数据为 CSV。项目体量：单个 HTML 文件即可运行。

AI 会给你一个总结。确认无误后回复：“阶段1确认，进入阶段2”。

在 TK-5 进度表里勾掉阶段1。

## 第 3 步：架构设计（阶段 2 简化版，5 分钟）

> 阶段2：请给出这个记账应用的业务流程描述，然后锁定目录结构（就一个 index.html 文件）和依赖（不需要任何外部依赖，纯 HTML+CSS+JS）。

AI 会输出流程描述和确认。你回复：“阶段2确认并锁定，进入阶段4（跳过阶段3，因为我们是单文件项目）”。

## 第 4 步：手动完成空项目初始化（阶段 4，2 分钟）

在你的 `my-account-book` 文件夹里，新建一个文件：
- Windows：右键 → 新建 → 文本文档，改名为 `index.html`
- Mac：用文本编辑新建文件，保存为 `index.html`

用 VS Code 或记事本打开这个文件，里面先空着。

告诉 AI：“阶段4完成，空 index.html 文件已创建在 my-account-book 文件夹中。进入阶段5。第一个模块：HTML 基本结构和 CSS 样式框架。”

## 第 5 步：增量编码（阶段 5，分 4 轮，每轮约 10 分钟）

**核心规则**：每次只让 AI 做一件事。你复制代码 → 粘贴到 `index.html` → 保存 → 用浏览器打开测试 → 确认正常 → 再让 AI 做下一件。

### 模块 5-1：HTML 结构和样式

> 阶段5第一个模块：先写 HTML 基本结构和 CSS 样式。让页面有一个好看的布局，包含：顶部标题栏（"个人记账本"）、中间余额显示区（大号数字）、下方添加记录的表单（类型选择：收入/支出、金额输入框、分类选择、备注输入框、添加按钮）、最下方记录列表区（表格形式，列：日期、类型、分类、金额、备注、删除按钮）。此时功能先不实现，只做静态页面。配色简洁，浅灰背景，白色卡片，蓝色强调色。

AI 生成代码后，全选复制 → 粘贴到 `index.html` → 保存 → 双击 `index.html` 用浏览器打开。
确认：页面结构完整，样式好看，按钮和输入框都正常显示。

### 模块 5-2：添加记录功能

> 阶段5第二个模块：加入 JavaScript 逻辑。实现添加记录功能：用户填写表单点击"添加"后，在记录列表中添加一行，同时更新余额。收入增加余额，支出减少余额。添加后清空表单。先不用本地存储。

复制新代码替换 `index.html` 全部内容 → 保存 → 浏览器刷新 → 测试添加几条收入和支出，确认余额正确变化。

### 模块 5-3：删除记录功能

> 阶段5第三个模块：实现删除记录功能。每条记录后面有一个删除按钮，点击后删除该行，同时余额反向更新（删除收入则减去，删除支出则加回）。删除前弹窗确认"确定要删除这条记录吗？"

同上流程：复制 → 粘贴 → 保存 → 刷新 → 测试。确认删除后余额正确，取消删除不影响。

### 模块 5-4：本地存储

> 阶段5第四个模块：实现 localStorage 本地存储。每次添加或删除记录时，自动保存数据到 localStorage。页面加载时自动读取 localStorage 恢复数据。这样刷新页面数据不会丢失。

同上流程测试。确认：添加记录后关闭浏览器再打开，数据还在。

## 第 6 步：测试与修复（阶段 6，10 分钟）

> 阶段6：请为我的记账应用生成一套测试清单。我需要逐项手动测试，发现问题反馈给你修复。测试场景需包括：
> 1. 正常添加收入和支出
> 2. 添加金额为 0 的记录
> 3. 添加金额为负数的记录
> 4. 不填金额直接点添加
> 5. 连续添加多条记录，余额累计是否正确
> 6. 删除一条记录后余额是否正确
> 7. 删除所有记录后余额是否归零
> 8. 刷新页面数据是否还在
> 9. 清空浏览器缓存后，页面显示空列表且余额为 0

AI 会给你一份清单。逐项测试，发现 Bug 就把现象告诉 AI：

> 测试第 3 项时，输入负数金额，余额计算错误。请修复。

AI 修复后重新测试，直到全部通过。

## 第 7 步：完善与交付（阶段 7-8，10 分钟）

> 阶段7：请统一检查代码风格，确保颜色一致、按钮样式统一、字体大小协调。如果有重复的代码逻辑，请合并。

AI 给出优化版本后，替换测试。

> 阶段8：请为这个记账应用生成一个简单的 README 文件（中文），包含功能介绍、使用方法、技术说明（纯 HTML+CSS+JS，localStorage 存储）。

把 README 内容保存为 `readme.txt`，放在同一个文件夹里。

## 最终成果

你的 `my-account-book` 文件夹里现在有：
- `index.html`：完整的记账应用
- `readme.txt`：使用说明

双击 `index.html` 就能用。把整个文件夹发给别人，对方也能直接用。

## 常见问题

**Q: 我粘贴代码后网页长得乱七八糟？**
A: 检查有没有漏掉 `<style>` 标签里的 CSS 部分。把整个 HTML 文件内容复制给 AI，问它“页面布局混乱，请检查并修复”。

**Q: 功能不工作，点了按钮没反应？**
A: 用浏览器打开 `index.html` 后按 F12，点 Console 标签，看有没有红色错误。截图或复制错误文字发给 AI：“我运行你的代码后出现这个错误：[粘贴错误]，请修复。”

**Q: localStorage 是什么？为什么数据不会丢？**
A: 它是浏览器提供的一个小数据库，数据存在你自己的电脑上。只要不清理浏览器缓存，数据就会一直在。
