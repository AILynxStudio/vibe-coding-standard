# 案例5：低代码 Agent——批量生成班级通讯录页面

> **难度**：★★（需要注册 Dify 或 Coze 平台账号）
> **工具包**：TK-6（低代码Agent固化工作流）
> **你需要**：一个 Dify 账号（dify.ai，免费注册）或 Coze 账号（coze.com）
> **你将做出**：一个能批量生成多个班级通讯录页面的自动化工作流

## 前置准备

1. 注册 Dify（dify.ai）或 Coze（coze.com）账号。
2. 登录后创建一个新应用（Dify 选"文本生成应用"或"Agent"类型）。
3. 准备好 TK-6 的内容（第〇章工具包）。

## 第 1 步：配置 Agent 工作流（10 分钟）

在 Dify 或 Coze 的应用设置中，找到"系统提示词"或"Instructions"设置。

将 TK-6 的完整内容粘贴进去。这就是你的 Agent 的"大脑"——它会按照 8 阶段流程自动干活。

保存设置。

## 第 2 步：输入需求（5 分钟）

在对话输入框中，描述你的需求：

> 我需要生成一个班级通讯录管理页面。需求如下：
> 1. 展示班级名称、学生列表（姓名、性别、联系电话、家长姓名、家庭住址）
> 2. 支持搜索学生姓名
> 3. 支持添加、编辑、删除学生信息
> 4. 数据保存在本地存储中
> 5. 使用 React + Ant Design 组件库
> 6. 技术栈：React + TypeScript + Ant Design

Agent 会自动按照 TK-6 固化的工作流，依次完成：
- 阶段1：需求确认
- 阶段2：架构设计（目录结构、组件树）
- 阶段3：环境对齐（检查知识库）
- 阶段4：输出项目初始化指令
- 阶段5：逐个模块生成代码
- 阶段6：生成测试用例
- 阶段7：代码优化建议
- 阶段8：打包部署说明

## 第 3 步：审核每个阶段的产出（10 分钟）

Agent 会在每个阶段完成后等待你的确认。你只需要：
1. 看产出是否符合预期
2. 回复"确认"或提出修改意见
3. Agent 自动进入下一阶段

## 第 4 步：批量生成（10 分钟）

第一个班级的通讯录完成后，告诉 Agent：

> 用同样的结构，再生成 4 个班级的通讯录页面。班级名称分别是：三年级一班、三年级二班、四年级一班、五年级一班。每个班级的页面结构相同，只是班级名称和初始数据不同。

Agent 会批量生成代码，每个生成完自动检查。

## 第 5 步：导出和部署（5 分钟）

Agent 最后会给出：
- 完整的项目文件清单
- 本地运行步骤
- 打包部署命令

按步骤在本地运行或部署即可。

## 常见问题

**Q: Agent 跳过阶段直接生成代码？**
A: 在对话中说"你跳过了阶段 X，请先完成阶段 X 的产出并等待我确认。"

**Q: 生成的代码有重复？**
A: 告诉 Agent："阶段7检查发现代码有重复逻辑，请合并公共组件。"

**Q: 想要不同的样式？**
A: 在阶段5之前或期间说："把主题色改成绿色，表格样式改成简洁风格。"
