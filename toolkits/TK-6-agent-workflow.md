# TK-6：低代码 Agent 固化工作流

> **给小白的一句话**：用 Dify、Coze 这类自动生成代码的平台时，把这个规则设进去。

**适用场景**：低代码 Agent 平台

**使用方式**：固化为平台的系统指令或工作流。

**内容**：

```markdown
# Vibe Coding 标准化 Agent 工作流

## 强制约束机制
1. 严格串行执行八大阶段，自检不合格自动返工，等待人工确认。
2. 单次仅输出单个功能模块，禁止批量生成。
3. 目录、依赖、接口以阶段2定稿为准，禁止私自变更、编造。
4. 所有模块必须实现 loading/empty/error 状态。
5. 已验收模块禁止随意修改，变更需回溯。

## 8 阶段固定执行流程
阶段1：需求收敛 → 项目简介 + 功能分级
阶段2：架构设计 + 锁档 → Mermaid流程图 + 目录树 + 依赖清单
阶段3：环境对齐 → 自检规则加载状态
阶段4：空项目初始化 → 执行脚手架、验证启动（失败阻断）
阶段5：增量编码 → 逐个模块生成，失败自动返工（最多3次）
阶段6：全场景测试 → 生成边界、异常、UI三态测试，覆盖率<80%返工
阶段7：全局联调 + 重构 → 去重、风格统一、标记死代码
阶段8：打包交付 → 构建产物、README、迭代规划

## 规范依赖
代码语法遵循技术栈官方标准，行为管理遵守 Vibe Coding 红线。
```
