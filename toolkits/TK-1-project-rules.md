# TK-1：项目硬约束模板 (CLAUDE.md / .cursorrules)

> **给小白的一句话**：这份文件是给 AI 立规矩用的。在项目文件夹里新建 `.cursorrules` 和 `CLAUDE.md`，把内容贴进去。

**适用场景**：Cursor（场景一）、VSCode + AI 插件（场景二）

**使用方式**：
- Cursor：在项目根目录同时创建 `.cursorrules` 和 `CLAUDE.md`，内容相同。
- VSCode + AI 插件：仅创建 `CLAUDE.md`。

**文件内容**：

```markdown
# 项目最高强制规则

## 1. 目录与文件红线
1. 严格遵守阶段2定稿目录结构，禁止私自新建、移动、删除目录。
2. 遵循单一文件单一职责原则，源码、配置、静态资源、文档分区存放。

## 2. 依赖与接口红线
1. 仅使用阶段2确认的依赖包，禁止私自新增第三方依赖。
2. 禁止编造不存在的接口、函数、字段、服务地址。
3. 所有外部请求统一集中封装管理。

## 3. 开发行为铁律
1. 严格按照8阶段流程串行开发，不跳步。
2. 增量开发，单次仅开发一个最小功能模块。
3. 禁止批量生成全量代码。
4. 已验收模块禁止随意修改。
5. 所有用户输入、网络请求必须实现加载态、空态、异常处理。

## 4. Git 规范
功能测试通过后方可提交，提交备注清晰标注模块。

## 5. 交付规范
最终交付物包含：可运行项目、标准README、环境配置说明。

## 层级说明
- 开发流程遵循通用流程 Skill（`vibe-coding-std`），部署路径见 TK-2。
- 代码语法遵循 `skills` 目录下的官方技术栈 Skill。
```
