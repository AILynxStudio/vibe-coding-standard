# 场景选择决策树

生成一张从左到右展开的决策树图，1200×900px，白色背景。

顶部标题：决策树——我该选哪个场景？

画面最左侧一个圆角矩形（深蓝 #1a365d 填充，白色文字）：你的需求是？

从该节点向右分出 3 条箭头分支：

上分支（通往右上）：
- 菱形判断框（浅灰填充，黑色文字）：完全不写代码，快速出原型？
- 向右箭头指向圆角结果框（绿色 #38a169 填充，白色文字）：场景五——无代码生成类

中分支（水平向右）：
- 菱形判断框：需要写代码，但不想配环境？
- 向右箭头指向结果框（橙色 #dd6b20 填充，白色文字）：场景三——桌面客户端类

下分支（通往右下）：
- 菱形判断框：需要公司级规范统一？
- 向右箭头指向结果框（紫色 #805ad5 填充，白色文字）：场景七——企业级认知类

从中分支的结果框继续向下分叉：
- 左分支：愿意换新 IDE，追求最强 AI？→ 场景一——智能IDE类（深蓝填充）
- 中分支：用现有 VSCode，不想改变习惯？→ 场景二——插件增强类（中蓝填充）
- 右分支：需要批量产出标准化模板？→ 场景四——低代码Agent类（灰蓝填充）

从场景一/二的节点继续向下分叉：
- 判断框：想尝试多 Agent 并行开发？
- 是 → 场景六——多智能体协作类（红色 #e53e3e 填充）
- 否 → 回到场景一或二

所有判断框使用菱形，结果框使用圆角矩形，箭头标注"是/否"或说明文字。
