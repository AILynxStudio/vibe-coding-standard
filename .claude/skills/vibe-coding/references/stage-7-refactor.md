## 代码重构

### 重构原则

```
1. 单一职责原则：每个函数/组件只做一件事
2. DRY原则：不要重复自己
3. KISS原则：保持简单
4. YAGNI原则：不要过度设计
```

### 重构清单


## 常见问题

### Q: lint检查失败怎么办？
A: 分析错误，修复后重新检查：
```bash
# 查看详细错误
npm run lint 2>&1 | tee lint-error.log

# 自动修复
npm run lint:fix

# 手动修复无法自动修复的问题
```

### Q: 性能问题怎么办？
A: 分析性能瓶颈，针对性优化：
```
1. 使用性能分析工具
2. 找到性能瓶颈
3. 针对性优化
4. 验证优化效果
```

### Q: 代码重复怎么办？
A: 提取公共函数/组件：
```
1. 识别重复代码
2. 提取公共函数/组件
3. 替换重复代码
4. 验证功能正常
```
