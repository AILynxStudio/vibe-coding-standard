# 第十章 场景六：多智能体协作类

## 10.1 场景定位
- **核心定义**：多个 AI 智能体扮演不同角色协作开发。
- **能力等级**：实验-高级
- **适用项目**：复杂系统、并行子任务、技术预研。
- **核心优势**：并行处理，内建审查。
- **核心限制**：通信可能出错，成本高，调试难。

## 10.2 代表工具
AutoGen, AgentStack, ChatDev.

## 10.3 8 阶段适配
人类为项目经理，将阶段任务分派：
- 阶段 1-2：架构师 Agent，人类审核锁定。
- 阶段 3-4：DevOps Agent。
- 阶段 5-6：多个程序员 Agent 并行，测试员 Agent 统一测试。
- 阶段 7-8：集成 Agent、文档 Agent。

## 10.4 7 铁律落地
接口定义 (OpenAPI/Protobuf) 是唯一真实来源，审查 Agent 检查铁律执行。

## 10.5 风险与补救
- 通信混乱 → 强类型接口契约。
- 成本高 → 仅在阶段 5-6 启用多 Agent 并行。
