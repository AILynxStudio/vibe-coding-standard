# 第一章 体系总纲

## 1.1 场景筛选规则

工业级开发对工具的**性能、稳定性、工程能力**有极高要求。以下类型工具被永久剔除：
- 在线 IDE（如 Replit、CodeSandbox Web 版）
- CLI 终端 AI 代理（纯命令行 Agent）
- 云开发环境（如 GitHub Codespaces 移动端）
- 实验性、频繁崩溃或输出不稳定的工具

仅保留具备**本地高性能环境、成熟文件管理、稳定 AI 交互界面**的工具。

## 1.2 场景职能划分

1. **工程开发场景**（可独立完成完整项目落地）：
   - 智能 IDE 类
   - 插件增强类
   - 桌面 AI 客户端类
   - 低代码 Agent 平台类
   - 无代码生成类
   - 多智能体协作类
   - 企业级认知类

2. **辅助咨询场景**（禁止完整项目编码落地）：网页大模型（ChatGPT、Claude 网页版等）。

## 1.3 四层解耦架构（优先级从高至低）

1. **项目硬约束层**（最高优先级） - `CLAUDE.md` / `.cursorrules`  
   管控目录结构、依赖、开发行为、Git 规范、交付标准。不包含代码语法。

2. **通用流程 Skill 层** - `vibe-coding-std`  
   统一 8 阶段流程、人机协作规则。

3. **技术规范层** - 官网/GitHub 原版 Skill  
   承载编码语法、框架规范。

4. **场景兜底层** - 专用提示词/固化工作流  
   无本地文件时的兜底方案。例如：网页大模型使用 TK-3 提示词，低代码平台固化 TK-6 工作流。

上层约束下层，规则出现歧义时以更高层级为准。
