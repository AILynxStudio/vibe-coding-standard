# 第六章 场景二：插件增强类

## 6.1 场景定位
- **核心定义**：传统 IDE (VS Code, JetBrains) + AI 插件，AI 作为副驾驶。
- **能力等级**：高级
- **适用项目**：日常开发、中小型项目、后端服务。
- **核心优势**：与现有工作流无缝集成，人保有最高控制权。
- **核心限制**：完全依赖人工驱动流程，规则需手动加载。

## 6.2 代表工具
| 工具 | 特点 |
|------|------|
| VS Code + Claude Code 插件 | 强代码库理解 |
| VS Code + Copilot | 内联补全 |
| JetBrains AI Assistant | 强类型语言深度分析 |

## 6.3 8 阶段落地
- 阶段切换完全由人工下令。
- 阶段 4 由人工在终端执行，成功后再告知 AI。
- 阶段 5 人工手动提供精确上下文，一次只给一个任务。
- 阶段 6 人工运行测试，反馈报错给 AI 修复。

## 6.4 7 铁律落地
- 禁止批量：人工一次只给一个任务。
- 禁止编造：要求 AI 先列接口清单，人工核实。
- 强制兜底：Code Review 重点查三态。

## 6.5 风险与补救
- 人容易跳步 → 项目管理工具固化 8 阶段 Task。
- AI 不自动读规则 → 每次对话首条粘贴 CLAUDE.md 和 Skill。
- 上下文丢失 → 长任务开启新对话。
