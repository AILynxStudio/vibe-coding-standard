# TK-5：8 阶段 Task 模板

> **给小白的一句话**：这是一个"任务清单"，抄在本子上或录入飞书、Notion，做完一步勾一步。

**适用场景**：所有工程开发场景

**使用方式**：导入项目管理工具或保存为 `VIBE-CODING-TASKS.md`。

**模板内容**：

```markdown
项目名称：[项目名] - Vibe Coding 开发

□ 阶段1：需求收敛 & 功能分级

- 产出：100字内项目简介
- 产出：核心/次要/扩展功能清单
- 门禁：人工确认

□ 阶段2：架构设计 + 目录 + 依赖定稿（核心锁档）

- 产出：Mermaid 业务流程图
- 产出：固定目录树
- 产出：依赖清单
- 门禁：人工确认并锁定
- 审计：锁档文件提交 Git

□ 阶段3：Skill 与规则环境对齐校验

- 确认：规则文件已加载、增量模式已理解
- 门禁：AI 复述规则，人工确认

□ 阶段4：空项目初始化（硬性门槛）

- 初始化脚手架，安装依赖，清理冗余
- 验证：空项目启动无报错
- 门禁：截图留证，人工确认

□ 阶段5：单模块增量编码

- 逐个模块开发，每次一个，提交 Git
- ⛔ 红线：禁止多模块并行

□ 阶段6：单模块全场景测试 & Bug 修复

- 覆盖正常、异常、边界、UI 三态
- 门禁：测试覆盖率 ≥80%

□ 阶段7：全局联调 + 代码重构 + 性能优化

- 全链路联调、统一风格、清理冗余
- 门禁：全量测试通过

□ 阶段8：打包交付 + 文档归档 + 迭代规划

- 项目打包、生成 README、制定迭代计划
- 门禁：最终交付物齐全

需求变更回溯：阶段5-7若需修改阶段2锁定内容，必须回到阶段2重新定稿并向下传播。
```

**导入各工具的注意事项**：
- Linear：直接粘贴 Markdown，勾选框被自动识别为 Task。
- Jira：转为 CSV（两列：Summary, Description），批量导入。
- Notion：粘贴后选中全部，使用"Turn into" → "To-do list"。
- 飞书多维表格：每阶段一行，"门禁"设为状态列。
- 直接使用：保存为 `VIBE-CODING-TASKS.md`，在编辑器中逐项打钩。
