# 需求模板

## 概述

需求模板用于阶段1：需求收敛，帮助用户整理项目需求。

## 模板内容

```markdown
# 项目需求文档

## 项目简介
[用<=100字描述项目用途、目标用户、核心痛点]

## 功能分级

### 核心功能（必须有）
1. [功能1]
2. [功能2]
3. [功能3]

### 次要功能（应该有）
1. [功能1]
2. [功能2]

### 扩展功能（可以有）
1. [功能1]
2. [功能2]

## 项目体量
- 规模：[小型/中型/大型]
- 预计开发时间：[X小时/X天/X周]
- 预计代码量：[X行/X文件]

## 运行载体
- 平台：[Web/移动端/桌面/CLI]
- 浏览器：[Chrome/Firefox/Safari/Edge]
- 设备：[PC/平板/手机]

## 技术约束
- 技术栈：[React/Vue/Python/Java/Go]
- 数据库：[MySQL/PostgreSQL/MongoDB/无]
- 第三方服务：[无/具体服务]

## 验收标准
- [ ] 核心功能正常工作
- [ ] 性能满足要求
- [ ] 安全性满足要求
- [ ] 兼容性满足要求
```

## 使用说明

1. 复制模板内容
2. 根据项目情况填写
3. 与AI确认需求
4. 人工审核后锁定

## 注意事项

1. 项目简介要简洁明了
2. 功能分级要合理
3. 项目体量要准确
4. 运行载体要明确
5. 技术约束要清晰
