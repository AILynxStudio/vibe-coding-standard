# 任务模板

## 概述

任务模板用于8阶段流程的进度跟踪，帮助用户管理项目进度。

## 模板内容

```markdown
# Vibe Coding 8阶段Task模板

## 项目信息
- 项目名称：[项目名称]
- 开始时间：[开始时间]
- 预计完成：[预计完成时间]

## 阶段1：需求收敛
- [ ] 项目目标清晰（<=100字）
- [ ] 功能分级合理（核心/次要/扩展）
- [ ] 项目体量评估准确
- [ ] 运行载体确认
- 确认人：_________ 确认时间：_________

## 阶段2：架构锁档
- [ ] 业务流程图已绘制（Mermaid格式）
- [ ] 目录结构已锁定
- [ ] 依赖清单已确认
- [ ] 接口定义已确认
- [ ] 同意锁定以上内容
- 确认人：_________ 确认时间：_________

## 阶段3：环境对齐
- [ ] CLAUDE.md / .cursorrules 已创建
- [ ] TK-1 内容已写入规则文件
- [ ] TK-2 已部署到正确位置
- [ ] 增量开发模式已理解
- 确认人：_________ 确认时间：_________

## 阶段4：脚手架初始化
- [ ] 项目初始化成功
- [ ] 依赖安装完成
- [ ] 冗余代码已清理
- [ ] 空项目启动无报错
- 确认人：_________ 确认时间：_________

## 阶段5：增量编码
- [ ] 单模块开发，未批量生成
- [ ] 未编造接口
- [ ] 已实现loading/empty/error状态
- [ ] 未修改已验收模块
- [ ] 未编写死代码
- 确认人：_________ 确认时间：_________

## 阶段6：测试验证
- [ ] 正常流程测试通过
- [ ] 空数据状态测试通过
- [ ] 非法输入测试通过
- [ ] 超时处理测试通过
- [ ] 断连处理测试通过
- [ ] Loading状态测试通过
- [ ] Error状态测试通过
- [ ] Empty状态测试通过
- [ ] 异常测试覆盖率≥80%
- 确认人：_________ 确认时间：_________

## 阶段7：重构优化
- [ ] 全业务链路联调通过
- [ ] 代码风格统一
- [ ] 冗余代码已清理
- [ ] 性能优化完成
- [ ] lint检查通过
- [ ] 全量测试通过
- 确认人：_________ 确认时间：_________

## 阶段8：交付归档
- [ ] 项目已成功打包
- [ ] README已生成
- [ ] 迭代规划已制定
- [ ] 代码已提交到Git
- 确认人：_________ 确认时间：_________

## 项目总结
- 实际完成时间：[实际完成时间]
- 总开发时间：[总开发时间]
- 遇到的问题：[遇到的问题]
- 解决方案：[解决方案]
- 经验教训：[经验教训]
```

## 使用说明

1. 复制模板内容
2. 根据项目情况填写
3. 每阶段完成后打钩
4. 记录确认人和确认时间

## 注意事项

1. 每阶段必须人工确认
2. 确认人和确认时间必须记录
3. 遇到问题及时记录
4. 项目完成后总结经验
