# 第二层：通用流程Skill层

## 概述

通用流程Skill层是四层架构中的第二层，负责统一8阶段流程、人机协作规则。

## 载体

- **Cursor**：`.cursor/skills/vibe-coding-std/index.md`
- **VSCode**：`skills/vibe-coding-std/index.md`

## 核心职责

1. 驱动8阶段串行执行
2. 每阶段产出需人工确认
3. 阶段门禁检查

## 8阶段流程定义

### 阶段1：需求收敛

**核心任务**：
- 用<=100字描述项目用途、目标用户、核心痛点
- 划分核心/次要/扩展功能
- 判定项目体量与运行载体

**产出**：
- 极简项目简介
- 功能分级清单
- 项目体量评估
- 运行载体确认

**门禁**：人工确认

### 阶段2：架构锁档（核心锁档）

**核心任务**：
- 绘制Mermaid业务流程图
- 完成模块拆分、定义接口
- 锁定最终目录结构
- 整理第三方依赖/SDK/Skill清单

**产出**：
- 流程图 + 架构说明 + 固定目录树 + 依赖清单
- 定稿后无重大需求变更不得修改

**门禁**：人工确认并锁定

### 阶段3：环境对齐

**核心任务**：
- 验证CLAUDE.md/.cursorrules是否加载
- 验证通用流程Skill和技术栈Skill就位
- 确认增量开发模式已被理解

**产出**：校验通过说明

**门禁**：人工确认

### 阶段4：脚手架初始化（硬性门槛）

**核心任务**：
- 根据依赖清单初始化项目
- 安装依赖
- 清理所有冗余示例代码
- 空项目必须成功启动且无报错

**产出**：纯净可运行的项目骨架

**门禁**：空项目启动成功，人工确认

### 阶段5：单模块增量编码

**核心任务**：
- 以单个最小功能模块为单位开发
- 每次仅生成当前模块代码，立即本地运行验证
- 禁止：批量生成、多模块并行、全量代码生成

**产出**：通过验证的单个功能模块

**门禁**：人工确认

### 阶段6：单模块全场景测试

**核心任务**：
- 覆盖正常流程、空数据、非法输入、超时、断连等异常场景
- 前端组件必须实现loading/empty/error状态
- 修复所有Bug

**产出**：具备完整容错能力的稳定模块

**门禁**：测试覆盖率≥80%，人工确认

### 阶段7：全局联调 + 代码重构 + 性能优化

**核心任务**：
- 全业务链路联调、统一代码风格
- 消除冗余代码/死代码、性能优化

**产出**：风格统一、运行稳定的可交付项目

**门禁**：lint检查通过，全量测试通过，人工确认

### 阶段8：打包交付 + 文档归档 + 迭代规划

**核心任务**：
- 按目标平台打包
- 生成标准README
- 制定下一版本迭代计划

**产出**：可运行制品、项目文档、迭代规划

**门禁**：人工确认

## 执行铁律

1. 八大阶段必须串行，上一阶段人工确认后方可进入下一阶段
2. 每阶段结束输出【阶段总结 + 下一阶段待办】

## 四重执行保障

- **AI被规则管住**：铁律写入CLAUDE.md / .cursorrules
- **人被流程管住**：8阶段Task模板逐项确认
- **过程被Git管住**：锁档文件、启动截图提交审计
- **质量被门禁管住**：测试覆盖率>=80%、lint通过
