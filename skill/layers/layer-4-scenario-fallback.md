# 第四层：场景兜底层

## 概述

场景兜底层是四层架构中的最低优先级层，为无法使用本地文件系统的工具环境提供降级方案。

## 载体

- **TK-3**：网页大模型专用提示词
- **TK-4**：桌面客户端专用提示词
- **TK-6**：低代码Agent固化工作流

## 核心职责

1. 无本地文件时的兜底方案
2. 通过提示词将规则注入AI上下文
3. 提供简化版的8阶段流程

## 使用条件

**判断标准**：工具是否支持本地文件系统监听

```
支持本地文件监听？
├── 是 → 使用L1/L2/L3，不需要L4
│   ├── Cursor → skill自动加载到上下文
│   ├── VSCode + AI插件 → skill手动加载
│   └── Windsurf/Trae → skill自动加载
│
└── 否 → 使用L4兜底方案
    ├── 网页版AI（ChatGPT/Claude网页版）→ TK-3
    ├── 桌面AI客户端（ChatGPT/Claude桌面版）→ TK-4
    ├── 低代码平台（Dify/Coze）→ TK-6
    └── 多智能体平台（AutoGen/AgentStack）→ TK-7
```

## TK-3：网页大模型专用提示词

### 定位

- 仅用于咨询，禁止编码落地
- 角色：顾问，不是执行者

### 使用场景

- 需求梳理咨询
- 架构设计咨询
- 技术选型咨询
- 代码审查咨询

### 核心规则

```
1. 不负责完整项目编码、工程搭建与批量代码编写
2. 仅提供咨询建议，不直接生成代码
3. 需求梳理、架构设计、技术选型等咨询工作
```

## TK-4：桌面客户端专用提示词

### 定位

- 完整规则集，每次对话粘贴
- 角色：执行者，但需人工驱动

### 使用场景

- 桌面AI客户端开发
- 无文件监听的环境
- 需要完整规则但无法自动加载

### 核心规则

```
1. 每次重要对话开头完整粘贴规则
2. 阶段1-3在客户端内对话完成，产出拷贝到本地
3. 阶段4完全由人工本地执行
4. 阶段5每次只描述一个模块，附上当前文件内容
5. 阶段6-8人工主控
```

### 部署方式

```bash

```

## TK-6：低代码Agent固化工作流

### 定位

- 将8阶段固化为平台工作流
- 角色：自动化执行者

### 使用场景

- Dify/Coze等低代码平台
- 需要全自动流转
- 标准化模板、批量产出

### 核心规则

```
1. 将8阶段固化为平台工作流
2. Agent必须串行执行
3. 阶段4失败阻断
4. 阶段5-6逐模块生成并自动测试
5. 失败自动返工（最多3次）
6. 覆盖率<80%返工
```

## 风险控制

### L4的局限性

| 限制类型 | 具体表现 |
|----------|----------|
| **规则加载方式** | 手动粘贴或平台固化，无法自动生效 |
| **流程控制** | 依赖人工驱动，无法自动串行 |
| **文件操作** | 无法直接读写本地文件，需人工搬运 |
| **终端执行** | 无法自动执行命令，需人工在本地操作 |
| **上下文保持** | 对话轮次增多后易丢失上下文 |
| **规则完整性** | 提示词有长度限制，无法承载全部规则 |

### 风险控制机制

**1. 强制行为准则（写入提示词）**
```
1. 禁止批量生成多模块代码，每次仅处理单个最小功能模块
2. 严禁编造任何不存在的接口、函数、字段
3. 必须包含加载态、空态、错误态
4. 无明确指令不得修改已验收模块
```

**2. 阶段确认机制**
```
每个阶段完成需输出【阶段总结 + 下一阶段待办】，等待确认后再继续
```

**3. 低代码平台的阻断逻辑（TK-6）**
```
阶段4 失败阻断
阶段5 失败自动返工（最多3次）
阶段6 覆盖率<80%返工
```

**4. 多智能体的角色隔离（TK-7）**
```
只负责分配的模块，不得越权
接口定义以阶段2锁定 schema 为准
```

## 升级路径

```
L4（兜底方案）→ L1/L2/L3（标准流程）

升级条件：
- 工具支持本地文件监听
- 用户需要更高的自动化程度
- 项目需要更严格的规则控制

升级方式：
- 安装skill到本地文件系统
- 部署TK-1和TK-2
- 使用标准8阶段流程
```
