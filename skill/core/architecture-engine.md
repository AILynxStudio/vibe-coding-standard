# 四层架构引擎

## 概述

四层架构引擎是Vibe Coding智能引导式SKILL的核心组件，负责管理和执行四层解耦架构的规则优先级。

## 架构定义

```
L1: 项目硬约束层（最高优先级）── .cursorrules / CLAUDE.md
L2: 通用流程Skill层 ──────────── vibe-coding-std（本skill）
L3: 技术规范层 ──────────────── React/Vue/Python等官方Skill
L4: 场景兜底层（最低优先级）── TK-3/TK-4/TK-6等
```

## 核心裁决规则

> **上层约束下层，规则出现歧义时以更高层级为准。**

## 执行流程

```
用户启动SKILL
    ↓
加载第四层：场景兜底规则（基础保障）
    ↓
加载第三层：技术规范（按需加载）
    ↓
加载第二层：通用流程Skill（8阶段驱动）
    ↓
加载第一层：项目硬约束（最高强制）
    ↓
四层规则融合，生成完整约束集
```

## 各层执行细节

### 第一层：项目硬约束层执行

**启动时**：
1. 检测是否存在 .cursorrules / CLAUDE.md
2. 如不存在，引导用户创建
3. 加载硬约束规则

**执行时**：
- AI生成代码前，检查是否违反硬约束
- 违反时主动拒绝执行
- 输出违反的具体规则和建议

**硬约束清单**：
1. 目录结构：严格遵守阶段2定稿目录
2. 依赖管理：仅使用阶段2确认的依赖
3. 开发行为：8阶段串行、增量开发
4. Git规范：功能测试通过后方可提交
5. 交付规范：可运行项目、标准README

### 第二层：通用流程Skill层执行

**加载时机**：
- 用户启动SKILL时自动加载
- 每阶段开始时重新加载当前阶段规则

**执行方式**：
- 驱动8阶段串行执行
- 每阶段产出需人工确认
- 阶段门禁检查

### 第三层：技术规范层执行

**加载时机**：
- 按需加载，根据项目技术栈选择

**执行方式**：
- 编码语法遵循技术栈Skill
- 技术实现细节由L3指导
- 不得突破L1/L2的约束边界

### 第四层：场景兜底层执行

**使用条件**：
- 仅当工具无法自动加载本地规则文件时使用

**执行方式**：
- TK-3：通过系统提示词限制AI只做顾问不写代码
- TK-4：每次对话开头重述核心规则
- TK-6：将8阶段和铁律固化为平台内置检查规则

## 冲突解决机制

### 四步判断法

```
步骤1: 判断是否有"铁律"被违反？
  ├── 是 → 立即停止执行，回滚，报告给人类 → END
  └── 否 → 进入步骤2

步骤2: 判断当前执行处于哪个阶段？
  ├── 阶段2已锁定的内容被改动？
  │   ├── 是 → 阻断，要求回到阶段2重新定稿 → END
  │   └── 否 → 进入步骤3
  └── 阶段4空项目初始化失败？
      ├── 是 → 绝对禁止进入阶段5 → END
      └── 否 → 进入步骤3

步骤3: 判断规则冲突来自哪个层级？
  ├── L1(硬约束) 与 L3(技术规范) 冲突 → L1 优先
  ├── L2(流程规则) 与 L3(技术规范) 冲突 → L2 优先
  ├── L1(硬约束) 与 L2(流程规则) 冲突 → L1 优先
  ├── L4(兜底) 与任意上层冲突 → 上层优先
  └── 同一层级内冲突 → 进入步骤4

步骤4: 无法自动裁决 → 报告给人类，由人类仲裁
```

### 冲突场景处理

**场景一：L1与L3的规范冲突**
- 表现：项目硬约束禁止某种依赖，但技术栈Skill推荐该依赖
- 裁决：L1胜出，AI不得引入未确认的依赖

**场景二：L2流程规则与L3技术实现的冲突**
- 表现：L2要求增量开发，但L3脚手架命令会一次生成多个文件
- 裁决：阶段4允许批量生成，阶段5必须回归增量模式

**场景三：L4兜底方案与上层规则的冲突**
- 表现：使用TK-3作为兜底时，无法实现L1/L2要求的本地文件监听
- 裁决：L4本身被设计为"咨询顾问"角色，从架构层面规避冲突

**场景四：多智能体间的技术规范冲突**
- 表现：多个AI Agent对同一接口产生不同理解
- 裁决：阶段2锁定的接口定义是"唯一真实来源"，人类拥有最终决定权

## 冲突解决的四个层次

| 层次 | 机制 | 适用场景 | 工具支撑 |
|------|------|----------|----------|
| **预防层** | 场景隔离 + TK文件互斥 | 避免规则冲突发生 | case-guide对照表 |
| **检测层** | AI自检 + CI门禁 | 规则违反的即时发现 | CLAUDE.md + GitHub Actions |
| **裁决层** | 层级优先级 + 人类仲裁 | 规则歧义时的决策 | 阶段确认机制 |
| **恢复层** | Git回滚 + 阶段回溯 | 冲突发生后的修复 | Git + Task模板 |
