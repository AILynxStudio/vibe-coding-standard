# 对话引导引擎

## 概述

对话引导引擎负责引导用户完成8阶段开发流程，通过AI主动提问和用户确认的方式推进项目。

## 交互流程概览

```
用户启动SKILL
    ↓
AI: 欢迎语 + 功能介绍
    ↓
用户: 描述项目需求
    ↓
AI: 提问5个关键问题
    ↓
用户: 回答问题
    ↓
AI: 分析并推荐配置
    ↓
用户: 确认配置
    ↓
AI: 引导执行8阶段流程
    ↓
每阶段结束：AI产出 + 人工确认
    ↓
项目完成
```

## 欢迎语设计

```
你好！我是Vibe Coding智能引导助手。

我可以帮你：
- 分析项目需求
- 选择合适的开发场景
- 按8阶段流程引导你完成项目开发

请问你想要开发什么项目？
```

## 阶段1：需求收敛引导

### AI引导话术

```
让我们从需求开始。我会问您几个问题来帮助梳理需求。

基于Vibe Coding的最佳实践，我需要了解：

1. **项目目标**：这个项目要解决什么问题？
2. **目标用户**：谁会使用这个项目？
3. **核心功能**：最核心的功能是什么？（最多3个）
4. **次要功能**：有哪些次要功能？
5. **扩展功能**：有哪些扩展功能？（可选）
6. **项目体量**：小型/中型/大型？
7. **运行载体**：Web/移动端/桌面/CLI？

请回答这些问题，我会帮您整理成需求文档。
```

### 产出

- 极简项目简介（<=100字）
- 功能分级清单（核心/次要/扩展）
- 项目体量评估
- 运行载体确认

### 门禁检查

```
AI: 我已经整理了您的需求：
[需求文档内容]

请确认：
- [ ] 项目目标清晰
- [ ] 功能分级合理
- [ ] 项目体量评估准确
- [ ] 运行载体确认

请确认是否准确？如有需要修改的地方请告诉我。
```

## 阶段2：架构锁档引导

### AI引导话术

```
现在进入架构设计阶段。这是核心锁档阶段，一旦确认，后续阶段不得私自修改。

基于您的需求，我会：

1. 绘制业务流程图（Mermaid格式）
2. 设计模块拆分方案
3. 定义接口规范
4. 锁定目录结构
5. 整理依赖清单

请告诉我：
1. 您偏好的技术栈是什么？（如：React/Vue/Python等）
2. 有必须使用的第三方服务吗？
3. 有性能或安全方面的特殊要求吗？
```

### 产出

- Mermaid业务流程图
- 架构说明文档
- 固定目录树
- 依赖清单

### 锁档确认

```
AI: 架构设计已完成。这是核心锁档文件：
[架构文档内容]

⚠️ 重要提醒：这是阶段2的核心锁档，一旦确认，后续阶段不得私自修改。

请确认：
- [ ] 我已审核业务流程图
- [ ] 我已确认目录结构
- [ ] 我已确认依赖清单
- [ ] 我已确认接口定义
- [ ] 我同意锁定以上内容

请确认是否同意锁定？
```

## 阶段3：环境对齐引导

### AI引导话术

```
现在检查开发环境是否就绪：

1. 检查您的AI工具是否已配置规则文件
2. 验证TK-1（项目硬约束）是否已加载
3. 确认TK-2（通用流程Skill）是否就位
4. 确认您理解增量开发模式

请告诉我您使用的AI工具，我会帮您配置。
```

### 检查清单

- [ ] CLAUDE.md / .cursorrules 已创建
- [ ] TK-1 内容已写入规则文件
- [ ] TK-2 已部署到正确位置
- [ ] 增量开发模式已理解

## 阶段4：脚手架初始化引导

### AI引导话术

```
现在初始化项目脚手架：

1. 根据依赖清单初始化项目
2. 安装所有必要依赖
3. 清理冗余示例代码
4. 验证空项目能否成功启动

请运行以下命令：
[初始化命令]

然后告诉我运行结果。
```

### 硬性门槛

```
AI: ⚠️ 硬性门槛：空项目必须成功启动且无报错，否则禁止进入编码阶段。

请确认：
- [ ] 项目初始化成功
- [ ] 依赖安装完成
- [ ] 冗余代码已清理
- [ ] 空项目启动无报错

请确认项目已成功启动？
```

## 阶段5：增量编码引导

### AI引导话术

```
现在开始编码。我们将采用增量开发模式：

本次开发的模块：[模块名称]
模块功能：[功能描述]
预计代码量：[行数]

我会：
1. 只生成当前模块的代码
2. 生成后立即本地运行验证
3. 验证通过后再开发下一个模块

⚠️ 禁止：批量生成、多模块并行、全量代码生成

请确认开始开发这个模块？
```

### 编码节奏

```
AI → 生成单模块代码
用户 → 运行验证
用户 → 确认通过
AI → 开发下一个模块
```

## 阶段6：测试验证引导

### AI引导话术

```
现在对模块进行全场景测试：

测试清单：
1. 正常流程测试
2. 空数据状态测试
3. 非法输入测试
4. 超时处理测试
5. 断连处理测试
6. Loading状态测试
7. Error状态测试
8. Empty状态测试

请逐项测试并告诉我结果。
```

### 测试报告

```
AI: 测试报告：
- 正常流程：✅ 通过
- 空数据状态：✅ 通过
- 非法输入：❌ 失败 - [问题描述]
- ...

需要修复的问题：[问题列表]
```

请运行以下命令检查代码质量：

### 检查清单

- [ ] 冗余代码已清理
- [ ] 性能优化完成

- [ ] 全量测试通过

## 阶段8：交付归档引导

### AI引导话术

```
项目即将完成！现在进行交付准备：

1. 按目标平台打包
2. 生成标准README
3. 制定迭代规划

请告诉我：
1. 目标平台是什么？（Web/移动端/桌面）
2. 需要部署到哪里？（GitHub Pages/Vercel/自有服务器）
3. 下一版本计划做什么功能？
```

### 交付清单

- [ ] 项目已成功打包
- [ ] README已生成
- [ ] 迭代规划已制定
- [ ] 代码已提交到Git

## 用户可能的回复及AI应对策略

### 阶段1：需求收敛

| 用户回复 | AI应对策略 |
|----------|-----------|
| **清晰描述需求** | 整理成需求文档，进入阶段2 |
| **模糊描述需求** | 追问具体问题："您说的XX具体指什么？" |
| **不确定需求** | 提供参考案例："您可以参考case-03的待办清单项目" |
| **需求过多** | 引导分级："请告诉我最核心的3个功能是什么？" |
| **需求过少** | 追问细节："除了XX功能，还有其他需求吗？" |

### 阶段2：架构锁档

| 用户回复 | AI应对策略 |
|----------|-----------|
| **确认架构方案** | 锁定架构，进入阶段3 |
| **对技术栈有疑问** | 解释技术栈优缺点，提供选择建议 |
| **想要修改架构** | 引导回到阶段2重新定稿 |
| **不确定技术选择** | 提供推荐方案："基于您的需求，推荐使用React" |

### 阶段3：环境对齐

| 用户回复 | AI应对策略 |
|----------|-----------|
| **环境已配置** | 验证配置，进入阶段4 |
| **不知道如何配置** | 提供详细步骤："请按以下步骤配置..." |
| **配置失败** | 诊断问题，提供解决方案 |
| **使用不同工具** | 调整配置方案，适配用户工具 |

### 阶段4：脚手架初始化

| 用户回复 | AI应对策略 |
|----------|-----------|
| **初始化成功** | 进入阶段5 |
| **初始化失败** | 诊断错误，提供修复方案 |
| **依赖安装失败** | 检查网络/版本，提供替代方案 |
| **空项目启动报错** | 分析错误日志，引导修复 |

### 阶段5：增量编码

| 用户回复 | AI应对策略 |
|----------|-----------|
| **模块开发完成** | 验证后进入下一个模块 |
| **代码有错误** | 分析错误，提供修复方案 |
| **想要修改需求** | 评估影响，必要时回到阶段2 |
| **AI批量生成** | 立即停止，回滚到上一次提交 |

### 阶段6：测试验证

| 用户回复 | AI应对策略 |
|----------|-----------|
| **测试通过** | 进入阶段7 |
| **测试失败** | 分析失败原因，引导修复 |
| **不知道如何测试** | 提供测试清单和步骤 |
| **覆盖率不足** | 补充测试用例 |

### 阶段7：重构优化

| 用户回复 | AI应对策略 |
|----------|-----------|
| **重构完成** | 进入阶段8 |
| **lint检查失败** | 分析错误，引导修复 |
| **性能问题** | 提供优化建议 |
| **代码重复** | 引导合并公共组件 |

### 阶段8：交付归档

| 用户回复 | AI应对策略 |
|----------|-----------|
| **打包成功** | 生成README，完成项目 |
| **打包失败** | 分析错误，提供修复方案 |
| **需要部署** | 提供部署指南 |
| **需要迭代规划** | 制定下一版本计划 |

## 异常情况处理

### 用户中途退出

```
AI: 检测到您中途退出。当前进度已保存。
下次启动时，我会从上次中断的阶段继续。
```

### 用户想要回退阶段

```
AI: 您想要回退到哪个阶段？
回退会影响后续阶段的产出，请确认。
```

### 用户修改需求

```
AI: 检测到需求变更。这会影响阶段2的锁档内容。
请确认是否回到阶段2重新定稿？
```
