# 质量门禁引擎

## 概述

质量门禁引擎负责在每个阶段设置质量门禁，确保项目质量可控。

## 质量保障闭环

```
需求 → 阶段2锁档 → 阶段4硬性门槛 → 阶段6测试覆盖 → 阶段7质量门禁 → 交付
  ↓         ↓              ↓                ↓                ↓           ↓
人工审核   Git审计        启动验证         覆盖率≥80%        全量测试     文档归档
```

## 各阶段门禁设计

### 阶段1：需求收敛门禁

**门禁条件**：
- [ ] 项目目标清晰（<=100字）
- [ ] 功能分级合理（核心/次要/扩展）
- [ ] 项目体量评估准确
- [ ] 运行载体确认

**检查方式**：人工审核

### 阶段2：架构锁档门禁

**门禁条件**：
- [ ] 业务流程图已绘制（Mermaid格式）
- [ ] 目录结构已锁定
- [ ] 依赖清单已确认
- [ ] 接口定义已确认
- [ ] 同意锁定以上内容

**检查方式**：人工审核 + Git审计

**锁档确认**：
```
AI: 架构设计已完成。这是核心锁档文件：
[架构文档内容]

⚠️ 重要提醒：这是阶段2的核心锁档，一旦确认，后续阶段不得私自修改。

请确认：
- [ ] 我已审核业务流程图
- [ ] 我已确认目录结构
- [ ] 我已确认依赖清单
- [ ] 我已确认接口定义
- [ ] 我同意锁定以上内容
```

### 阶段3：环境对齐门禁

**门禁条件**：
- [ ] CLAUDE.md / .cursorrules 已创建
- [ ] TK-1 内容已写入规则文件
- [ ] TK-2 已部署到正确位置
- [ ] 增量开发模式已理解

**检查方式**：人工验证

### 阶段4：脚手架初始化门禁（硬性门槛）

**门禁条件**：
- [ ] 项目初始化成功
- [ ] 依赖安装完成
- [ ] 冗余代码已清理
- [ ] 空项目启动无报错

**检查方式**：人工验证 + Git审计

**硬性门槛**：
```
AI: ⚠️ 硬性门槛：空项目必须成功启动且无报错，否则禁止进入编码阶段。

请确认：
- [ ] 项目初始化成功
- [ ] 依赖安装完成
- [ ] 冗余代码已清理
- [ ] 空项目启动无报错
```

**铁律检查**：铁律6（空项目初始化失败，绝对不允许编写业务代码）

### 阶段5：增量编码门禁

**门禁条件**：
- [ ] 单模块开发，未批量生成
- [ ] 未编造接口
- [ ] 已实现loading/empty/error状态
- [ ] 未修改已验收模块
- [ ] 未编写死代码

**检查方式**：人工验证 + Git审计

**铁律检查**：
- 铁律1：单模块开发，未批量生成
- 铁律3：未编造接口
- 铁律4：已实现loading/empty/error状态
- 铁律5：未修改已验收模块
- 铁律7：未编写死代码

### 阶段6：测试验证门禁

**门禁条件**：
- [ ] 正常流程测试通过
- [ ] 空数据状态测试通过
- [ ] 非法输入测试通过
- [ ] 超时处理测试通过
- [ ] 断连处理测试通过
- [ ] Loading状态测试通过
- [ ] Error状态测试通过
- [ ] Empty状态测试通过
- [ ] 异常测试覆盖率≥80%

**检查方式**：人工测试 + 覆盖率报告

**铁律检查**：
- 铁律4：异常场景全覆盖
- 铁律5：修复未引入新Bug

### 阶段7：重构优化门禁

**门禁条件**：
- [ ] 全业务链路联调通过
- [ ] 代码风格统一
- [ ] 冗余代码已清理
- [ ] 性能优化完成
- [ ] lint检查通过
- [ ] 全量测试通过

**检查方式**：lint检查 + 全量测试

**铁律检查**：
- 铁律7：已清理冗余代码
- 铁律5：重构未破坏已有功能

### 阶段8：交付归档门禁

**门禁条件**：
- [ ] 项目已成功打包
- [ ] README已生成
- [ ] 迭代规划已制定
- [ ] 代码已提交到Git

**检查方式**：人工验证 + Git审计

## 质量门禁执行机制

### 门禁检查流程

```
阶段完成
    ↓
AI生成门禁检查清单
    ↓
用户逐项检查
    ├── 全部通过 → 进入下一阶段
    └── 部分失败 → 修复后重新检查
```

### 门禁失败处理

**处理流程**：
1. 识别失败项
2. 分析失败原因
3. 提供修复方案
4. 修复后重新检查
5. 全部通过后才允许继续

**示例**：
```
AI: 测试报告：
- 正常流程：✅ 通过
- 空数据状态：✅ 通过
- 非法输入：❌ 失败 - [问题描述]
- ...

需要修复的问题：[问题列表]

请修复后重新测试。
```

## 质量指标

### 代码质量指标

| 指标 | 目标值 | 检查阶段 |
|------|--------|----------|
| 异常测试覆盖率 | ≥80% | 阶段6 |
| lint检查通过率 | 100% | 阶段7 |
| 全量测试通过率 | 100% | 阶段7 |
| 代码重复率 | <5% | 阶段7 |

### 流程质量指标

| 指标 | 目标值 | 检查阶段 |
|------|--------|----------|
| 阶段门禁通过率 | 100% | 每阶段 |
| 铁律违反次数 | 0 | 每阶段 |
| 需求变更次数 | 最小化 | 全流程 |
| 回滚次数 | 最小化 | 全流程 |
