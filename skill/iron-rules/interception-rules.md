# 拦截规则清单

## 概述

拦截规则清单定义了在开发过程中需要拦截的具体规则和处理方式。

## 拦截规则分类

### 1. 铁律拦截（绝对禁止）

| 铁律 | 拦截条件 | 拦截动作 | 提示信息 |
|------|----------|----------|----------|
| 铁律1 | 批量生成多模块 | 阻断+回滚 | "请单模块开发" |
| 铁律2 | 修改锁档文件 | 阻断 | "锁档文件不可修改" |
| 铁律3 | 编造不存在的接口 | 阻断+拒绝 | "请先确认接口存在" |
| 铁律6 | 空项目启动失败 | 阻断 | "禁止编写业务代码" |

### 2. 质量拦截（条件禁止）

| 铁律 | 拦截条件 | 拦截动作 | 提示信息 |
|------|----------|----------|----------|
| 铁律4 | 缺少状态处理 | 阻断+补充 | "请补充loading/empty/error状态" |
| 铁律5 | 修改已验收模块 | 阻断+确认 | "已验收模块不可修改，是否需要变更？" |
| 铁律7 | 包含死代码 | 警告+清理 | "请清理死代码" |

### 3. 流程拦截（阶段门禁）

| 阶段 | 拦截条件 | 拦截动作 | 提示信息 |
|------|----------|----------|----------|
| 阶段2 | 未确认锁档 | 阻断 | "请确认架构锁档" |
| 阶段3 | 环境未对齐 | 阻断 | "请完成环境配置" |
| 阶段4 | 空项目启动失败 | 阻断 | "请修复初始化问题" |
| 阶段5 | 批量生成 | 阻断+回滚 | "请单模块开发" |
| 阶段6 | 测试覆盖率不足 | 阻断 | "请补充测试用例" |
| 阶段7 | lint检查失败 | 阻断 | "请修复代码风格问题" |

## 拦截规则详细说明

### 铁律1拦截：批量生成多模块

**拦截条件**：
- AI一次生成多个文件
- AI生成全量代码
- AI并行开发多个模块

**拦截动作**：
1. 立即停止AI生成
2. 回滚到上一次提交
3. 只保留目标模块的改动
4. 重新引导AI只生成单个模块

**提示信息**：
```
⚠️ 铁律1违反：检测到批量生成多模块
请单模块开发，每次只处理一个最小功能模块。

恢复步骤：
1. 回滚到上一次提交：git reset --hard HEAD~1
2. 重新引导AI只生成单个模块
```

### 铁律2拦截：修改锁档文件

**拦截条件**：
- 修改阶段2锁定的目录结构
- 修改阶段2锁定的依赖清单
- 修改阶段2锁定的接口定义

**拦截动作**：
1. 阻断修改
2. 提示需要回到阶段2重新定稿
3. 记录修改需求

**提示信息**：
```
⚠️ 铁律2违反：检测到修改阶段2锁档内容
锁档文件不可修改，如需变更请回到阶段2重新定稿。

变更流程：
1. 记录变更需求
2. 回到阶段2重新定稿
3. 向下传播所有变更
4. 重新执行后续阶段
```

### 铁律3拦截：编造不存在的接口

**拦截条件**：
- 调用不存在的API
- 调用不存在的函数
- 使用不存在的依赖包

**拦截动作**：
1. 阻断生成
2. 拒绝使用编造的接口
3. 要求先确认接口存在

**提示信息**：
```
⚠️ 铁律3违反：检测到编造不存在的接口
请先确认接口存在，再进行编码。

验证步骤：
1. 要求AI输出接口清单
2. 人工核实接口是否存在
3. 确认后才允许编码
```

### 铁律4拦截：缺少完整状态处理

**拦截条件**：
- 网络请求缺少loading状态
- 数据展示缺少empty状态
- 用户输入缺少异常处理

**拦截动作**：
1. 阻断继续
2. 提示补充状态处理
3. 提供补充方案

**提示信息**：
```
⚠️ 铁律4违反：缺少完整状态处理
请补充loading/empty/error状态。

补充清单：
- [ ] 网络请求添加loading状态
- [ ] 数据展示添加empty状态
- [ ] 用户输入添加异常处理
- [ ] 错误场景添加error状态
```

### 铁律5拦截：修改已验收模块

**拦截条件**：
- 修改已测试通过的模块
- 修改已提交Git的模块
- 修改已人工确认的模块

**拦截动作**：
1. 阻断修改
2. 提示需要正式变更流程
3. 记录变更需求

**提示信息**：
```
⚠️ 铁律5违反：检测到修改已验收模块
已验收模块不可随意修改，如需变更请走正式流程。

变更流程：
1. 记录变更需求
2. 评估变更影响
3. 人工审批变更
4. 重新验收模块
```

### 铁律6拦截：空项目初始化失败

**拦截条件**：
- 空项目启动报错
- 依赖安装失败
- 环境配置错误

**拦截动作**：
1. 绝对阻断
2. 禁止进入阶段5
3. 要求修复初始化问题

**提示信息**：
```
⚠️ 铁律6违反：空项目初始化失败
绝对禁止编写业务代码，请先修复初始化问题。

修复步骤：
1. 分析错误日志
2. 修复初始化问题
3. 重新验证空项目启动
4. 成功后才允许继续
```

### 铁律7拦截：包含死代码

**拦截条件**：
- 包含TODO注释
- 包含空函数体
- 包含注释掉的代码
- 包含永远不会执行的分支

**拦截动作**：
1. 警告提醒
2. 提示清理死代码
3. 提供清理方案

**提示信息**：
```
⚠️ 铁律7警告：检测到死代码
请清理死代码，保持代码纯净。

清理清单：
- [ ] 移除TODO注释
- [ ] 实现或移除空函数体
- [ ] 移除注释掉的代码
- [ ] 移除永远不会执行的分支
```

## 拦截规则执行流程

```
代码生成/修改
    ↓
检查拦截规则
    ├── 铁律拦截 → 绝对阻断 → 回滚/拒绝
    ├── 质量拦截 → 条件阻断 → 补充/确认
    ├── 流程拦截 → 阶段门禁 → 修复/重试
    └── 无拦截 → 允许继续
```

## 拦截规则配置

### 启用/禁用规则

```yaml
interception_rules:
  iron_rule_1: enabled  # 批量生成拦截
  iron_rule_2: enabled  # 锁档修改拦截
  iron_rule_3: enabled  # 编造接口拦截
  iron_rule_4: enabled  # 状态处理拦截
  iron_rule_5: enabled  # 已验收修改拦截
  iron_rule_6: enabled  # 空项目拦截
  iron_rule_7: enabled  # 死代码拦截
```

### 规则严重程度

```yaml
severity_levels:
  iron_rule_1: error    # 绝对禁止
  iron_rule_2: error    # 绝对禁止
  iron_rule_3: error    # 绝对禁止
  iron_rule_4: warning  # 条件禁止
  iron_rule_5: warning  # 条件禁止
  iron_rule_6: error    # 绝对禁止
  iron_rule_7: warning  # 条件禁止
```
