# 下篇：七场景适配总览

## 能力矩阵

| 场景大类 | 能力等级 | 规则生效方式 | 流程控制 | 适用项目 |
|----------|:--------:|--------------|----------|----------|
| 智能IDE类 | 顶级 | 自动加载 | Agent辅助+人工确认 | 复杂全栈、正式上线 |
| 插件增强类 | 高级 | 手动/自动 | 人工全程 | 日常开发、中小型项目 |
| 桌面客户端类 | 中级 | 手动导入 | 人工全程 | 原型验证、一次性脚本 |
| 低代码Agent类 | 自动化 | 平台固化 | 全自动 | 标准化模板、批量产出 |
| 无代码生成类 | 非工程 | 不适用 | 人工描述迭代 | 原型/文档、简单工具 |
| 多智能体协作类 | 实验-高级 | 注入系统提示 | Agent间协作+人工监督 | 复杂系统、技术预研 |
| 企业级认知类 | 组织级 | 企业级Ruleset | CI/CD门禁 | 多团队协作、遗留迁移 |

## 决策树
- 不写代码 → 场景五
- 写代码但不想配环境 → 场景三
- 公司级统一 → 场景七
- 换新 IDE → 场景一
- 保留 VSCode → 场景二
- 批量产出 → 场景四
- 多 Agent 并行 → 场景六

## 规则加载路径
- 智能IDE类：自动监听 `.cursorrules`
- 插件增强类：手动指定或首条粘贴 `CLAUDE.md`
- 桌面客户端：每轮对话粘贴规则
- 低代码平台：平台固化
- 企业级：CI 注入组织级 Ruleset

## 场景升级路径
- 原型→正式：场景五 → 场景三/场景一
- 个人→团队→企业：场景二 → 场景一 → 场景七
- 手工→批量→组织：场景四 → 场景七
不可逆向下沉。
