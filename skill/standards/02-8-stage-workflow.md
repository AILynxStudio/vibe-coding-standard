# 第二章 全局 8 阶段开发流程

所有工程场景必须严格执行，严禁跳步。每阶段完成需输出总结并由人工确认。

## 阶段 1：需求收敛 & 功能分级
- 用 ≤100 字描述项目用途、目标用户、核心痛点。
- 划分核心/次要/扩展功能。
- 判定项目体量与运行载体。
产出：极简项目简介 + 功能分级清单。

## 阶段 2：架构设计 + 目录 + 依赖定稿（核心锁档）
- 绘制 Mermaid 业务流程图。
- 完成模块拆分，定义接口。
- 锁定最终目录结构。
- 整理第三方依赖、SDK、Skill 清单。
产出：流程图 + 架构说明 + 固定目录树 + 依赖清单。
⚠️ 定稿后无重大需求变更不得修改，变更需人工回溯。

## 阶段 3：Skill 与规则环境对齐校验
- 验证 `CLAUDE.md`/`.cursorrules` 是否加载。
- 验证通用流程 Skill 和技术栈 Skill 就位。
- 确认增量开发模式已被理解。
产出：校验通过说明。

## 阶段 4：项目脚手架初始化（硬性门槛）
- 根据依赖清单初始化项目，安装依赖。
- 清理所有冗余示例代码。
- **硬性要求**：空项目必须成功启动且无报错，否则禁止进入编码。
产出：纯净可运行的项目骨架。

## 阶段 5：单模块最小粒度增量编码
- 以单个最小功能模块为单位开发。
- 每次仅生成当前模块代码，立即本地运行验证。
- **禁止**：批量生成、多模块并行、全量代码生成。
产出：通过验证的单个功能模块。

## 阶段 6：单模块全场景测试 & Bug 修复
- 覆盖正常流程、空数据、非法输入、超时、断连等异常场景。
- 前端组件必须实现 loading / empty / error 状态。
- 修复所有 Bug。
产出：具备完整容错能力的稳定模块。

## 阶段 7：全局联调 + 代码重构 + 性能优化
- 全业务链路联调，统一代码风格。
- 消除冗余代码、死代码，性能优化。
产出：风格统一、运行稳定的可交付项目。

## 阶段 8：打包交付 + 文档归档 + 迭代规划
- 按目标平台打包。
- 生成标准 README，制定下一版本迭代计划。
产出：可运行制品、项目文档、迭代规划。

## 人机协作分工
- **人类（不可委托）**：架构决策、阶段审核、质量把关、安全合规。
- **AI（可委托）**：需求初稿、流程图、脚手架、编码、测试、文档。
