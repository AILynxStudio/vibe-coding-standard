# 工具包依赖关系

## 概述

工具包依赖关系定义了TK-1到TK-7之间的依赖关系和优先级。

## 工具包列表

| 编号 | 名称 | 所属层级 | 主要功能 |
|------|------|----------|----------|
| TK-1 | 项目硬约束模板 | L1 | 定义项目最高强制规则 |
| TK-2 | 通用流程Skill | L2 | 定义8阶段开发流水线 |
| TK-3 | 网页大模型提示词 | L4 | 网页版AI咨询 |
| TK-4 | 桌面客户端提示词 | L4 | 桌面版AI开发 |
| TK-5 | 8阶段Task模板 | L2 | 进度跟踪 |
| TK-6 | 低代码Agent固化工作流 | L4 | 平台固化工作流 |
| TK-7 | 多智能体指令集 | L2 | 多角色分工协作 |

## 依赖关系图

```
TK-1（项目硬约束）
    │
    ├──→ TK-2（通用流程）
    │        │
    │        ├──→ TK-5（任务模板）
    │        │
    │        ├──→ TK-6（低代码工作流）
    │        │
    │        └──→ TK-7（多智能体指令集）
    │
    └──→ TK-3（网页大模型提示词）
    │
    └──→ TK-4（桌面客户端提示词）
```

## 依赖规则

### 1. TK-1依赖TK-2

**依赖原因**：项目硬约束引用通用流程路径

**依赖内容**：
```
TK-1中包含：
- 开发流程遵循通用流程 Skill（`vibe-coding-std`），部署路径见 TK-2。
```

### 2. TK-5依赖TK-1/TK-2

**依赖原因**：Task模板基于8阶段流程

**依赖内容**：
```
TK-5中包含：
- 8阶段Task模板
- 每阶段门禁条件
- 确认清单
```

### 3. TK-6依赖TK-2

**依赖原因**：低代码工作流固化8阶段

**依赖内容**：
```
TK-6中包含：
- 8阶段固化为平台工作流
- 阶段4失败阻断
- 阶段5-6逐模块生成并自动测试
```

### 4. TK-7依赖TK-2

**依赖原因**：多智能体分工基于8阶段

**依赖内容**：
```
TK-7中包含：
- 阶段1-2：架构师Agent
- 阶段3-4：DevOps Agent
- 阶段5-6：多个程序员Agent并行
- 阶段7-8：集成Agent、文档Agent
```

## 优先级关系

### 层级优先级

```
L1（项目硬约束层）> L2（通用流程Skill层）> L3（技术规范层）> L4（场景兜底层）
```

### 工具包优先级

```
TK-1（L1）> TK-2（L2）> TK-5（L2）> TK-7（L2）> TK-3（L4）> TK-4（L4）> TK-6（L4）
```

## 冲突解决

### 同层级冲突

**表现**：同一层级内多个工具包冲突

**解决方式**：
1. 人工选择激活哪个工具包
2. 或按场景隔离使用

### 不同层级冲突

**表现**：不同层级工具包冲突

**解决方式**：
1. 上层约束下层
2. L1优先于L2
3. L2优先于L3
4. L3优先于L4

## 版本兼容性

### 向后兼容原则

```
1. 次版本升级必须向后兼容
2. 主版本升级可以不向后兼容
3. 废弃功能至少保留一个次版本
```

### 版本检查

```bash
# 检查当前版本
cat VERSION

# 检查是否有更新
git fetch --tags
git describe --tags --abbrev=0
```

## 工具包更新规则

### 更新触发条件

1. 工具包内容变更
2. 标准体系调整
3. 新增场景
4. 废弃场景

### 更新流程

```
1. 修改提案
2. 一致性检查
3. 版本号更新
4. 审核合并
5. 发布更新
```
