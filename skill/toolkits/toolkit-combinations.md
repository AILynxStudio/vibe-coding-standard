# 工具包组合推荐

## 概述

工具包组合推荐定义了不同场景下的最佳工具包组合。

## 核心依赖链

```
TK-1（项目硬约束）→ TK-2（通用流程）→ TK-5（任务模板）
     ↓                    ↓                    ↓
  规则定义            流程驱动            执行跟踪
```

## 场景-工具包映射表

| 场景 | 推荐组合 | 部署方式 | 自动化程度 |
|------|----------|----------|------------|
| 智能IDE | TK-1 + TK-2 + TK-5 | 文件部署 | 高 |
| 插件增强 | TK-1 + TK-2 + TK-5 | 文件部署+手动加载 | 中 |
| 桌面客户端 | TK-4 + TK-5 | 每次对话粘贴 | 低 |
| 低代码平台 | TK-6 | 平台固化 | 全自动 |
| 无代码生成 | TK-3 | 系统提示词 | 无 |
| 多智能体 | TK-7 + TK-5 | 角色分发 | 高 |
| 企业级 | TK-1(组织级) + TK-5(CI化) | CI/CD集成 | 高 |

## 详细组合说明

### 智能IDE场景（Cursor、Windsurf、Trae）

**推荐组合**：TK-1 + TK-2 + TK-5

**部署方式**：
1. TK-1 写入 .cursorrules 和 CLAUDE.md
2. TK-2 部署到 .cursor/skills/vibe-coding-std/
3. TK-5 作为Task模板使用

**自动化程度**：高（AI自动推进）

### 插件增强场景（VSCode + AI插件）

**推荐组合**：TK-1 + TK-2 + TK-5

**部署方式**：
1. TK-1 写入 CLAUDE.md
2. TK-2 部署到 skills/vibe-coding-std/
3. TK-5 作为Task模板，人工驱动

**自动化程度**：中（人工驱动）

### 桌面客户端场景（ChatGPT/Claude桌面版）

**推荐组合**：TK-4 + TK-5

**部署方式**：
1. TK-4 每次对话开头粘贴
2. TK-5 作为Task模板

**自动化程度**：低（全程手动）

### 低代码Agent场景（Dify、Coze）

**推荐组合**：TK-6

**部署方式**：
1. TK-6 固化为平台工作流

**自动化程度**：全自动

### 无代码生成场景（v0、Bolt.new）

**推荐组合**：TK-3（仅咨询）

**部署方式**：
1. 复制TK-3内容作为系统提示词

**自动化程度**：无

### 多智能体场景（AutoGen、AgentStack）

**推荐组合**：TK-7 + TK-5

**部署方式**：
1. TK-7 分发给各Agent
2. TK-5 作为统一Task模板

**自动化程度**：高（并行处理）

### 企业级场景（Sourcegraph Cody）

**推荐组合**：TK-1(组织级) + TK-5(CI化)

**部署方式**：
1. TK-1 写入组织级规则
2. TK-5 集成到CI/CD

**自动化程度**：高（CI驱动）

## 工具包依赖关系

### 依赖规则

1. TK-1依赖TK-2：项目硬约束引用通用流程路径
2. TK-5依赖TK-1/TK-2：Task模板基于8阶段流程
3. TK-6依赖TK-2：低代码工作流固化8阶段
4. TK-7依赖TK-2：多智能体分工基于8阶段

### 依赖图

```
TK-1 ──→ TK-2 ──→ TK-5
  │        │
  │        ↓
  └──────→ TK-6
  │
  └──────→ TK-7
```

## 自动组合逻辑

```
用户选择场景 → 自动推荐工具包组合 → 引导部署
```

## 版本管理

### 版本号规则

- 主版本号（X.0.0）：标准体系重大调整或场景增删
- 次版本号（0.X.0）：新增案例、工具包微调
- 修订号（0.0.X）：文字修正、格式调整

### 升级路径

```
当前版本：1.0.0

升级路径：
├── 修订号升级（1.0.0 → 1.0.1）
│   └── 触发：文字修正、格式调整
│   └── 影响：无功能变更
│
├── 次版本升级（1.0.0 → 1.1.0）
│   └── 触发：新增案例、工具包微调
│   └── 影响：新增功能，向后兼容
│
└── 主版本升级（1.0.0 → 2.0.0）
    └── 触发：标准体系重大调整
    └── 影响：可能不向后兼容
```
