# 场景三：桌面AI客户端类

## 概述

桌面AI客户端类场景开箱即用，无文件监听，全程手动。

## 特点

- **自动化程度**：低
- **规则加载方式**：每轮对话粘贴TK-4
- **人工介入程度**：全程手动
- **适用项目**：快速原型、中型项目

## 工具包组合

**推荐组合**：TK-4 + TK-5

**部署方式**：
1. TK-4 每次对话开头粘贴
2. TK-5 作为Task模板

## 执行要点

### 阶段1-3：对话完成

```
阶段1-3在客户端内对话完成，产出拷贝到本地：
1. 在桌面客户端完成需求梳理
2. 在桌面客户端完成架构设计
3. 在桌面客户端完成环境对齐
4. 将产出拷贝到本地文件
```

### 阶段4：脚手架初始化

```
阶段4完全由人工本地执行，通过后告知AI：
1. 人工在本地执行初始化命令
2. 人工确认空项目启动
3. 人工告知AI初始化成功
```

### 阶段5：增量编码

```
阶段5每次只描述一个模块，附上当前文件内容：
1. 人工描述当前模块需求
2. 附上当前文件内容
3. AI生成模块代码
4. 人工复制代码到本地
5. 人工验证代码
```

### 阶段6-8：人工主控

```
阶段6-8人工主控：
1. 人工运行测试
2. 人工重构优化
3. 人工打包交付
```

## 风险与补救

### 风险1：上下文丢失

**表现**：长对话后AI忘记之前的规则

**补救措施**：
```
1. 每轮开头重述核心规则
2. 使用外部检查清单强制执行阶段
3. 定期保存阶段性成果
```

### 风险2：代码搬运错误

**表现**：复制粘贴过程中可能出错

**补救措施**：
```
1. 仔细核对代码
2. 使用diff工具对比
3. 验证后提交Git
```

### 风险3：规则失效

**表现**：AI不自动读取规则

**补救措施**：
```
1. 每轮对话粘贴TK-4
2. 使用外部检查清单
3. 定期检查规则是否生效
```

## TK-4使用说明

### TK-4内容

```
桌面客户端专用提示词：
- 完整规则集
- 每次对话粘贴
- 角色：执行者，但需人工驱动
```

### 使用方式

```
每次重要对话开头完整粘贴规则：
1. 复制TK-4内容
2. 粘贴到对话开头
3. 等待AI确认
4. 开始对话
```

## 最佳实践

### 1. 规则重述

```
每轮对话开头，重述核心规则：
- 铁律清单
- 当前阶段要求
- 确认清单
```

### 2. 人工驱动

```
人工主导流程控制：
- 阶段切换由人工下令
- 验证由人工执行
- 确认由人工完成
```

### 3. 外部检查

```
使用外部工具检查：
- Task模板逐项确认
- 项目管理工具跟踪进度
- Git审计记录变更
```
