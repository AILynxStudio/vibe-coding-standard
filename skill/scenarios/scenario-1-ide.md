# 场景一：智能IDE类（Cursor、Windsurf、Trae）

## 概述

智能IDE类场景是自动化程度最高的场景，Agent可以自主推进项目开发。

## 特点

- **自动化程度**：高
- **规则加载方式**：自动加载.cursorrules和.cursor/skills/
- **人工介入程度**：低
- **适用项目**：复杂全栈应用

## 工具包组合

**推荐组合**：TK-1 + TK-2 + TK-5

**部署方式**：
1. TK-1 写入 .cursorrules 和 CLAUDE.md
2. TK-2 部署到 .cursor/skills/vibe-coding-std/
3. TK-5 作为Task模板使用

## 部署步骤

### 1. 部署TK-1（项目硬约束）

```bash
# 创建 .cursorrules 文件
cp TK-1-project-rules.md .cursorrules

# 创建 CLAUDE.md 文件
cp TK-1-project-rules.md CLAUDE.md
```

### 2. 部署TK-2（通用流程Skill）

```bash
# 创建skills目录
mkdir -p .cursor/skills/vibe-coding-std

# 复制Skill文件
cp TK-2-workflow-skill.md .cursor/skills/vibe-coding-std/index.md
```

### 3. 验证部署

```bash
# 检查文件是否存在
ls -la .cursorrules CLAUDE.md .cursor/skills/vibe-coding-std/index.md
```

## 执行要点

### 阶段2：架构锁档

```
锁档后立即写入.cursorrules红线：
1. 更新目录结构约束
2. 更新依赖清单约束
3. 更新接口定义约束
```

### 阶段4：脚手架初始化

```
Agent执行命令，人工确认空项目启动成功：
1. Agent执行初始化命令
2. 人工确认空项目启动
3. 截图留证，提交Git
```

### 阶段5：增量编码

```
给极精确指令：
- ✅ "仅开发UserLogin组件的前端部分"
- ❌ "开发用户登录功能"
```

### 阶段6：测试验证

```
要求AI生成全场景测试并在终端运行：
1. AI生成测试清单
2. AI执行测试
3. 人工确认测试结果
```

## 风险与补救

### 风险1：AI批量生成多模块

**表现**：AI一次生成多个文件

**补救措施**：
```
1. 开启所有"需人工批准"选项
2. 指令加严格限定词
3. 立即停止，回滚到上一次提交
```

### 风险2：AI修改锁档文件

**表现**：AI修改阶段2锁定的内容

**补救措施**：
```
1. 在.cursorrules中明确禁止修改锁档文件
2. 开启"需人工批准"选项
3. 发现修改立即回滚
```

### 风险3：AI编造不存在的接口

**表现**：AI调用不存在的API

**补救措施**：
```
1. 要求AI先输出接口清单
2. 人工核实接口是否存在
3. 确认后才允许编码
```

## 最佳实践

### 1. 精确指令

```
✅ 好的指令：
"仅开发Header组件，包含标题显示和用户头像功能，不包含导航菜单"

❌ 不好的指令：
"开发Header组件"
```

### 2. 阶段确认

```
每个阶段结束后，人工确认：
- 检查产出是否符合预期
- 确认是否进入下一阶段
- 记录确认时间
```

### 3. Git审计

```
每个模块完成后，提交Git：
git add .
git commit -m "[Stage-5] 模块：Header组件开发完成"
```
