# 场景五：无代码生成类（v0、Bolt.new）

## 概述

无代码生成类场景零门槛，极速生成，无法深度控制逻辑。

## 特点

- **自动化程度**：无
- **规则加载方式**：手动粘贴TK-3
- **人工介入程度**：全程手动
- **适用项目**：原型、内部小工具

## 工具包组合

**推荐组合**：TK-3（仅咨询）

**部署方式**：
1. 复制TK-3内容作为系统提示词

## 执行要点

### 阶段1-2：脑中规划

```
阶段1-2在脑中规划：
1. 明确项目需求
2. 设计简单架构
3. 不需要正式文档
```

### 阶段5：增量追加功能

```
阶段5改为增量追加功能：
1. 生成基础版本
2. 逐步追加功能
3. 每次追加后验证
```

### 阶段6：人工手动测试

```
阶段6人工手动测试：
1. 测试核心功能
2. 测试边界条件
3. 记录测试结果
```

### 阶段8：一键部署或导出

```
阶段8一键部署或导出：
1. 使用平台部署功能
2. 或导出代码到本地
3. 使用其他方式部署
```

## 风险与补救

### 风险1：无法深度控制逻辑

**表现**：平台生成的代码不符合预期

**补救措施**：
```
1. 验证后导出代码
2. 转入场景一或场景二
3. 使用标准流程开发
```

### 风险2：平台限制

**表现**：平台功能限制

**补救措施**：
```
1. 选择功能丰富的平台
2. 结合人工干预
3. 导出代码后定制开发
```

### 风险3：质量控制

**表现**：自动生成的代码质量不确定

**补救措施**：
```
1. 人工审核生成结果
2. 测试核心功能
3. 导出代码后优化
```

## TK-3使用说明

### TK-3内容

```
网页大模型专用提示词：
- 仅用于咨询，禁止编码落地
- 角色：顾问，不是执行者
- 使用场景：需求梳理、架构设计、技术选型、代码审查
```

### 使用方式

```
复制TK-3内容作为系统提示词：
1. 复制TK-3内容
2. 粘贴到系统提示词
3. 开始咨询对话
```

## 最佳实践

明确项目需求：
