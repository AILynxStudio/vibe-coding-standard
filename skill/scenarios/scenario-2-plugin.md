# 场景二：插件增强类（VS Code + AI插件）

## 概述

插件增强类场景与现有工作流集成，完全依赖人工驱动。

## 特点

- **自动化程度**：中
- **规则加载方式**：手动加载skills/目录
- **人工介入程度**：高
- **适用项目**：日常开发、中小型项目

## 工具包组合

**推荐组合**：TK-1 + TK-2 + TK-5

**部署方式**：
1. TK-1 写入 CLAUDE.md
2. TK-2 部署到 skills/vibe-coding-std/
3. TK-5 作为Task模板，人工驱动

## 部署步骤

### 1. 部署TK-1（项目硬约束）

```bash
# 创建 CLAUDE.md 文件
cp TK-1-project-rules.md CLAUDE.md
```

### 2. 部署TK-2（通用流程Skill）

```bash
# 创建skills目录
mkdir -p skills/vibe-coding-std

# 复制Skill文件
cp TK-2-workflow-skill.md skills/vibe-coding-std/index.md
```

### 3. 验证部署

```bash
# 检查文件是否存在
ls -la CLAUDE.md skills/vibe-coding-std/index.md
```

## 执行要点

### 阶段切换

```
阶段切换完全由人工下令：
1. 人工确认当前阶段完成
2. 人工下令进入下一阶段
3. AI按指令执行
```

### 阶段4：脚手架初始化

```
阶段4由人工在终端执行，成功后再告知AI：
1. 人工执行初始化命令
2. 人工确认空项目启动
3. 人工告知AI初始化成功
```

### 阶段5：增量编码

```
人工手动提供精确上下文，一次只给一个任务：
1. 人工描述当前模块需求
2. AI生成模块代码
3. 人工验证代码
4. 人工确认后进入下一个模块
```

### 阶段6：测试验证

```
人工运行测试，反馈报错给AI修复：
1. 人工运行测试
2. 人工记录测试结果
3. 如有失败，反馈给AI修复
4. 修复后重新测试
```

## 风险与补救

### 风险1：规则失效

**表现**：AI不自动读取规则

**补救措施**：
```
1. 每次对话首条粘贴规则
2. 项目管理工具固化8阶段Task
3. 定期检查规则是否生效
```

### 风险2：流程跳步

**表现**：人工容易跳过某些阶段

**补救措施**：
```
1. 使用Task模板逐项确认
2. 每阶段结束强制确认
3. 使用外部检查清单
```

### 风险3：上下文丢失

**表现**：长对话后AI忘记之前的规则

**补救措施**：
```
1. 每轮对话重述核心规则
2. 使用外部检查清单
3. 定期保存阶段性成果
```

## 最佳实践

### 1. 规则粘贴

```
每次对话开头，粘贴核心规则：
- 铁律清单
- 当前阶段要求
- 确认清单
```

### 2. 人工驱动

```
人工主导流程控制：
- 阶段切换由人工下令
- 验证由人工执行
- 确认由人工完成
```

### 3. 外部检查

```
使用外部工具检查：
- Task模板逐项确认
- 项目管理工具跟踪进度
- Git审计记录变更
```
