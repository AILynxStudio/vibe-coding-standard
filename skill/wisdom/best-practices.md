# 最佳实践

## 概述

最佳实践总结了Vibe Coding标准体系中的核心实践原则。

## 核心实践原则

### 1. 规则先行，代码后行

**原则**：在开始编码前，先部署规则文件

**深层原因**：
- AI的行为完全由上下文决定
- 规则文件是持久化的上下文
- 没有规则文件，AI会使用默认行为

**实践方式**：
```
1. 部署TK-1到.cursorrules/CLAUDE.md
2. 部署TK-2到skills目录
3. 验证规则文件已加载
4. 开始编码
```

### 2. 阶段确认，步步为营

**原则**：每阶段结束，人工确认后才能进入下一阶段

**深层原因**：
- 这是人机协作的契约机制
- 没有确认，AI会自动进入下一阶段
- 确认语句本身也是上下文的一部分

**实践方式**：
```
1. AI产出阶段成果
2. 人工审核产出
3. 人工确认通过
4. 进入下一阶段
```

### 3. 增量编码，单模块验证

**原则**：每次只开发一个最小功能模块

**深层原因**：
- AI的"幻觉"问题
- 生成的代码越多，出错概率越高
- 增量编码使得错误定位成本最低

**实践方式**：
```
1. 选择下一个模块
2. AI生成模块代码
3. 本地运行验证
4. 验证通过后提交Git
5. 重复以上步骤
```

### 4. 测试清单，逐项验证

**原则**：使用测试清单逐项验证

**深层原因**：
- 人类测试时容易遗漏边界情况
- AI生成的测试清单是系统性的
- 逐项测试的过程本身就是对需求的二次确认

**实践方式**：
```
1. AI生成测试清单
2. 人工逐项测试
3. 记录测试结果
4. 修复失败项
```

### 5. 错误反馈，精确描述

**原则**：把报错发给AI，比描述问题更有效

**深层原因**：
- AI无法"猜测"错误
- 需要精确的错误信息
- 精确的错误描述能让AI一次性修复

**实践方式**：
```
1. 遇到错误时，复制错误信息
2. 粘贴给AI
3. AI分析错误原因
4. AI提供修复方案
```

## 阶段实践

### 阶段1：需求收敛

**实践要点**：
- 用<=100字描述项目
- 划分核心/次要/扩展功能
- 判定项目体量与运行载体

**常见错误**：
- 需求描述过于模糊
- 功能分级不合理
- 项目体量评估不准确

### 阶段2：架构锁档

**实践要点**：
- 绘制Mermaid业务流程图
- 锁定目录结构
- 整理依赖清单
- 定义接口规范

**常见错误**：
- 架构设计不合理
- 目录结构不清晰
- 依赖清单不完整
- 接口定义不明确

### 阶段3：环境对齐

**实践要点**：
- 验证规则文件已加载
- 验证Skill已部署
- 确认增量开发模式已理解

**常见错误**：
- 规则文件未加载
- Skill未部署
- 增量开发模式未理解

### 阶段4：脚手架初始化

**实践要点**：
- 初始化项目
- 安装依赖
- 清理冗余代码
- 验证空项目启动

**常见错误**：
- 初始化失败
- 依赖安装失败
- 冗余代码未清理
- 空项目启动报错

### 阶段5：增量编码

**实践要点**：
- 单模块开发
- 本地运行验证
- 验证通过后提交Git

**常见错误**：
- 批量生成多模块
- 未验证就提交
- 修改已验收模块

### 阶段6：测试验证

**实践要点**：
- 全场景测试
- 测试覆盖率≥80%
- 修复所有Bug

**常见错误**：
- 测试场景不全面
- 覆盖率不足
- Bug未修复

### 阶段7：重构优化

**实践要点**：
- 全局联调
- 统一代码风格
- 清理冗余代码
- 性能优化

**常见错误**：
- 重构破坏已有功能
- 代码风格不统一
- 冗余代码未清理

### 阶段8：交付归档

**实践要点**：
- 打包交付
- 生成README
- 制定迭代规划

**常见错误**：
- 打包失败
- README不完整
- 迭代规划不清晰
