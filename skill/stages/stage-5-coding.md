# 阶段5：单模块增量编码

## 概述

阶段5负责按增量开发模式，逐个模块开发功能代码。

## 核心任务

1. 以单个最小功能模块为单位开发
2. 每次仅生成当前模块代码，立即本地运行验证
3. 禁止：批量生成、多模块并行、全量代码生成

## AI引导话术

```
现在开始编码。我们将采用增量开发模式：

本次开发的模块：[模块名称]
模块功能：[功能描述]
预计代码量：[行数]

我会：
1. 只生成当前模块的代码
2. 生成后立即本地运行验证
3. 验证通过后再开发下一个模块

⚠️ 禁止：批量生成、多模块并行、全量代码生成

请确认开始开发这个模块？
```

## 编码节奏

```
AI → 生成单模块代码
用户 → 运行验证
用户 → 确认通过
AI → 开发下一个模块
```

## 铁律检查

- 铁律1：单模块开发，未批量生成
- 铁律3：未编造接口
- 铁律4：已实现loading/empty/error状态
- 铁律5：未修改已验收模块
- 铁律7：未编写死代码

## 常见问题

### Q: AI批量生成多个文件怎么办？
A: 立即停止，回滚到上一次提交：
```bash
# 回滚到上一次提交
git reset --hard HEAD~1

# 重新引导AI只生成单个模块
```

### Q: 模块开发失败怎么办？
A: 分析错误，修复后重新验证：
```bash
# 查看错误信息
npm run dev 2>&1 | tee error.log

# 修复问题
# 重新验证
```

### Q: 需要修改已验收模块怎么办？
A: 走正式变更流程：
```
1. 记录变更需求
2. 评估变更影响
3. 人工审批变更
4. 重新验收模块
```
